# 英威腾 GD200A 变频器 RS485（ModBus RTU）开发指令速查

> 适用案例：`Bus04_ModBusRTU_Invt_GD200A`  
> 协议：ModBus RTU，功能码 03H / 06H / 10H  
> **单片机 UART 参数必须与 P14 组完全一致**

---

## 一、硬件接线

| 变频器端子 | 接法 | 说明 |
|-----------|------|------|
| 485+ | 接模块 A+ | ModBus A 线 |
| 485- | 接模块 B- | ModBus B 线 |
| SG / 0V | 接模块 GND | **必须共地** |

**注意：**

- 屏蔽双绞线，屏蔽层单端接地
- 长距离总线末端加 **120Ω** 终端电阻
- A/B 接反会导致无响应或 CRC 错误
- STM32：UART2 `PA2(TX)` / `PA3(RX)` → RS485 模块，自动方向

---

## 二、变频器参数配置

### 2.1 P14 通讯组（必配）

| 功能码 | 名称 | 本案例值 | 说明 |
|--------|------|----------|------|
| P14.00 | 本机通讯地址 | **1** | 1~247 |
| P14.01 | 通讯波特率 | **4（19200）** | 3=9600 / 4=19200 |
| P14.02 | 数据格式 | **1（8E1）** | 0=8N1 / 1=8E1 / 2=8O1 |
| P14.03 | 通讯应答延时 | 5 ms | 从站应答间隔 |
| P14.04 | 通讯超时故障 | 0.0 s | 测试阶段建议 0 |
| P14.05 | 传输错误处理 | 1 | 1=不报警继续 |
| P14.06 | 通讯处理动作 | 0x000 | 标准地址模式 |

**STM32 `board.h` 对应（本案例 19200 8E1）：**

```c
19200, USART_WordLength_9b, USART_StopBits_1, USART_Parity_Even
```

| P14.02 | 帧格式 | STM32 配置 |
|--------|--------|------------|
| 0 | 8N1 | `8b` + `No` |
| 1 | 8E1 | **`9b` + `Even`** |
| 2 | 8O1 | `9b` + `Odd` |

### 2.2 P00 组（写启停/设频必配）

| 功能码 | 名称 | 值 | 说明 |
|--------|------|-----|------|
| P00.01 | 运行指令通道 | **2** | 2=通讯 |
| P00.02 | 通讯运行指令通道 | 0 | 0=ModBus |
| P00.06 | A 频率指令选择 | **8** | 8=ModBus 设频 |

## 三、ModBus RTU 帧格式

| 段 | 说明 |
|----|------|
| 帧头 | ≥3.5 字符时间空闲 |
| 从机地址 | 1 字节 |
| 功能码 | 1 字节 |
| 数据域 | N 字节 |
| CRC | 2 字节（**低字节在前**） |

| 功能码 | 说明 |
|--------|------|
| 03H | 读保持寄存器 |
| 06H | 写单个寄存器 |
| 10H | 写多个寄存器 |

## 四、寄存器地址速查

### 4.1 通讯控制（写 06H，2000H）

| 地址 | 数据值 | 命令 |
|------|--------|------|
| 2000H | 0001H | 正转运行 |
| | 0002H | 反转运行 |
| | **0005H** | **停机（减速）** |
| | 0006H | 自由停机 |
| | 0007H | 故障复位 |

### 4.2 通讯设定（写/读，2001H 起）

| 地址 | 说明 | 比例 |
|------|------|------|
| **2001H** | 通讯设定频率 | **0.01 Hz**（25Hz→2500） |
| 2002H | PID 给定 | 1000=100.0% |

### 4.3 状态监控（读 03H）

| 地址 | 说明 | 数据意义 |
|------|------|----------|
| **2100H** | 状态字 1 | 0001:正转 / 0002:反转 / **0003:停机** / 0004:故障 |
| 2101H | 状态字 2 | Bit0:就绪；Bit5~6:控制源 |
| **2102H** | **故障代码** | **不是识别码** |
| **2103H** | **识别代码** | **GD200A = 0x0107** |

**状态字 2（2101H）常用位：**

| 位 | 含义 |
|----|------|
| Bit0 | 0=未就绪 / 1=运行准备就绪 |
| Bit1~2 | 电机号（0~3 → 电机 1~4） |
| Bit5~6 | 00=键盘 / 01=端子 / **10=通讯** |

### 4.4 运行数据监控（读 03H）

| 地址 | 说明 | 比例 / 备注 |
|------|------|-------------|
| **3000H** | 运行频率 | 0.01 Hz |
| 3001H | 设定频率 | 0.01 Hz |
| **3002H** | 母线电压 | **0.1 V**（5840→584.0V） |
| 3003H | 输出电压 | 1 V |
| 3004H | 输出电流 | 0.1 A |
| 3005H | 运行转速 | 1 RPM |
| 3006H | 输出功率 | 0.1%（有符号） |
| 3007H | 输出转矩 | 0.1%（有符号） |
| 3008H | 闭环给定 | 0.1% |
| 3009H | 闭环反馈 | 0.1% |
| 300AH | 输入端子 DI | 十六进制位图 |
| 300BH | 输出端子 DO | 十六进制位图 |
| 300CH~300FH | 模拟量 AI1~4 | 0.01 V（0~1000=0~10.00V） |
| 3010H | 高速脉冲输入 | 0.01 kHz |
| 3012H | 多段速当前段 | 0~15 |
| 3016H | 识别代码（备用） | 0x0107 |

**AI 通道无效值：** 原始 `0xFFFF` 或 `>1000` 表示未接/无效。

### 4.5 功能码参数地址

公式：`地址 = 组号 × 256 + 组内序号（十六进制）`

| 功能码 | 地址 | 说明 |
|--------|------|------|
| P00.01 | 0001H | 运行指令通道 |
| P14.00 | 0E00H | 通讯地址 |
| P14.01 | 0E01H | 波特率 |

## 五、RTU 指令示例（从站 01H，19200 8E1）

> 帧内容与波特率/校验无关；以下 CRC 已含。

### 5.1 读状态字 1（通讯测试推荐）

```text
发送: 01 03 21 00 00 01 8E 36
返回: 01 03 02 00 03 F8 45    ← 0003H=停机
```

### 5.2 读识别码 2103H

```text
发送: 01 03 21 03 00 01 6F F6
返回: 01 03 02 01 07 [CRC]    ← GD200A
```

### 5.3 读运行频率 3000H

```text
发送: 01 03 30 00 00 01 8B CA
返回: 01 03 02 13 88 [CRC]    ← 5000=50.00Hz
```

### 5.4 设频 25.00Hz + 正转

```text
写频率: 01 06 20 01 09 C4 [CRC]   ← 2500=25.00Hz
写正转: 01 06 20 00 00 01 43 CA
```

### 5.5 反转

```text
发送: 01 06 20 00 00 02 03 CA
```

### 5.6 停机

```text
发送: 01 06 20 00 00 05 82 09
```

### 5.7 同时写正转 + 50Hz（10H）

```text
发送: 01 10 20 00 00 02 04 00 01 13 88 3B 10
返回: 01 10 20 00 00 02 4C 0A
```

## 六、英威腾异常码（区别于标准 Modbus）

| 码 | 名称 | 含义 |
|----|------|------|
| 01H | 非法命令 | 功能码不支持 |
| 02H | 非法地址 | 寄存器地址错误 |
| 03H | 非法数据值 | 数据超出范围 |
| 04H | 操作失败 | 无效设置 |
| **06H** | **数据帧错误** | **帧长/CRC 错误（非“设备忙”）** |
| 07H | 参数只读 | 写了只读寄存器 |

异常应答格式：`功能码 | 0x80` + 异常码，如 `83 06` = FC03 异常 06H。

## 七、C 代码 API 示例

### 7.1 读运行频率

```c
uint16_t raw;
ModBusRTU_ReadHoldingRegisters(UART_INSTANCE_2, 1, 0x3000, 1, &raw, 1000);
/* raw / 100.0f → Hz */
```

### 7.2 设频 + 正转（先频后令）

```c
ModBusRTU_WriteSingleRegister(UART_INSTANCE_2, 1, 0x2001, 2500, 1000); /* 25Hz */
Delay_ms(80);
ModBusRTU_WriteSingleRegister(UART_INSTANCE_2, 1, 0x2000, 0x0001, 1000); /* 正转 */
```

### 7.3 停机

```c
ModBusRTU_WriteSingleRegister(UART_INSTANCE_2, 1, 0x2000, 0x0005, 1000);
```

### 7.4 批量读（本案例用法）

```c
uint16_t status_blk[4];   /* 2100H~2103H */
uint16_t mon_blk[16];       /* 3000H~300FH */
ModBusRTU_ReadHoldingRegisters(UART_INSTANCE_2, 1, 0x2100, 4, status_blk, 1000);
ModBusRTU_ReadHoldingRegisters(UART_INSTANCE_2, 1, 0x3000, 16, mon_blk, 1000);
```

## 八、案例自动测试流程（阶段 3）

本案例 `main_example.c` 上电后自动执行：

```text
1. 485 预检（读 2100H / 3000H 等）
2. 预停机（2000H=0005H）
3. 正转频率爬升：
   for freq = 25, 30, 35:
       写 2001H = freq × 100
       首档再写 2000H = 0001H，后续只写 2001H
       每档保持 5s（每 2s 打印简要状态）
4. 反转频率爬升：
   for freq = 25, 30, 35:
       写 2001H = freq × 100
       首档再写 2000H = 0002H，后续只写 2001H
       每档保持 5s
5. 写 2000H = 0005H 停机
6. 停机监控：每 2s 读全量寄存器，一参数一行输出
```

**可调宏（main_example.c）：**

| 宏 | 默认 | 说明 |
|----|------|------|
| `INVT_TEST_FREQ_START_HZ` | 25 | 起始频率 |
| `INVT_TEST_FREQ_END_HZ` | 35 | 结束频率 |
| `INVT_TEST_FREQ_STEP_HZ` | 5 | 步进 |
| `INVT_TEST_STEP_MS` | 5000 | 每档保持 ms |

## 九、常见故障排查

| 现象 | 可能原因 |
|------|----------|
| Timeout | 线参不一致；A/B 反；未共地；地址错 |
| BadRsp / Overrun | 8E1 未用 9b 字长；MCU 收包延时过长 |
| 异常 06H | 帧/CRC 错误（英威腾定义） |
| 能设频不运行 | P00.01≠2 或 P00.06≠8 |
| 母线 5839V 显示 | 比例错误；正确为 583.9V（÷10） |
| AI 655.35V | 0xFFFF 无效，应显示 N/A |

## 十、快速上电 Checklist

- [ ] 485+→A+，485-→B-，SG/GND 共地
- [ ] P14.00=1，P14.01=4，P14.02=1（19200 8E1）
- [ ] 读 2100H 返回 0003H（停机）等正常状态
- [ ] 读 2103H 返回 **0x0107**（勿读 2102H 当识别码）
- [ ] 阶段 3：**P00.01=2，P00.06=8**
- [ ] 写顺序：**先 2001H 设频 → 再 2000H 命令**
- [ ] 负载可安全正反转，人在旁监护
