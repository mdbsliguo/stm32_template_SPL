# MAX31856 最简单案例

## 简介

这是一个**极简版本**的MAX31856读取案例，去掉了所有复杂的错误处理、重试机制等，只保留最核心的功能，用于快速验证SPI通信和MAX31856配置。

## 📚 重要文档

**⚠️ 调试笔记**：请务必阅读 [`DEBUG_NOTES.md`](./DEBUG_NOTES.md)，其中详细记录了所有调试过程中发现的问题、原因分析和解决方案。这是理解为什么代码能正常工作的关键文档。

## 硬件连接

```
STM32F103C8T6    MAX31856模块
PB12 (CS)    ->  CS
PB13 (SCK)   ->  SCK
PB14 (MISO)  ->  SDO (MISO)
PB15 (MOSI)  ->  SDI (MOSI)
3.3V         ->  VCC (或3V)
GND          ->  GND
```

## 功能

1. **最简单的初始化**：直接初始化SPI2和MAX31856
2. **最简单的配置**：
   - K型热电偶
   - 连续转换模式
   - 4次平均（快速）
3. **直接读取**：每500ms读取一次温度并显示

## 使用方法

1. 编译并下载程序
2. 观察OLED显示：
   - 第1行：故障状态
   - 第2行：热电偶温度（TC）
   - 第3行：冷端温度（CJ）

## 注意事项

- 如果显示"ERR"，说明SPI通信失败，请检查硬件连接
- 确保所有SPI线都正确连接，特别是MISO（PB14）和CS（PB12）
- 确保GND共地
- 确保电源（3.3V）正确连接

## 与完整版本的区别

- **去掉**：复杂的错误处理、重试机制、数据有效性检查
- **保留**：核心的SPI通信、MAX31856配置、温度读取
- **目标**：快速验证硬件连接和基本功能

## 🔍 调试过程

本项目经过详细调试，解决了以下关键问题：

1. **SR寄存器地址**：0x0F（故障状态记录寄存器）
2. **热电偶温度寄存器地址**：0x0C-0x0E（LTCBH/LTCBM/LTCBL）
3. **冷端温度寄存器地址**：0x0A-0x0B（CJTH/CJTL）
4. **冷端温度数据格式**：12位有符号数，分辨率0.0625°C
5. **热电偶温度数据格式**：19位有符号数，分辨率0.0078125°C
6. **转换模式**：必须设置CMODE位（bit7）= 1启用连续转换
7. **SPI通信时序**：CS拉低后延时200μs，拉高前后各延时50μs
8. **SPI时钟频率**：2.25MHz（预分频32，不超过5MHz限制）

详细的问题分析和解决方案请参考 [`DEBUG_NOTES.md`](./DEBUG_NOTES.md)。
