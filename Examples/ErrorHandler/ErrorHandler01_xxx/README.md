# ErrorHandler01 - 错误处理模块测试

## 📋 案例目的

- **核心目标**：测试error_handler错误处理模块的完整功能，展示错误处理的完整流程
- **学习重点**：
  - 理解统一错误处理机制的工作原理
  - 掌握错误码的定义和使用方法
  - 学习错误处理函数的调用方式（ErrorHandler_Handle、ErrorHandler_Check）
  - 了解错误回调函数的注册和使用
  - 学习错误码到字符串的转换功能
  - 理解不同模块的错误处理方式
- **应用场景**：适用于需要统一错误处理的应用，如复杂系统、多模块项目等

## 🔧 硬件要求

### 必需外设

- **LED1**：`PA1`（错误提示，错误发生时闪烁）

### 传感器/模块

- **OLED显示屏**（SSD1306，I2C接口）
  - SCL：`PB8`
  - SDA：`PB9`
  - VCC：3.3V
  - GND：GND

| STM32F103C8T6 | 外设/模块 | 说明 |
|--------------|----------|------|
| PA1 | LED1正极 | LED1负极接GND |
| PB8 | OLED SCL | I2C时钟线 |
| PB9 | OLED SDA | I2C数据线 |
| 3.3V | OLED VCC | 电源 |
| GND | OLED GND | 地线 |

## 🔄 实现流程

### 整体逻辑

本案例通过6个测试阶段，全面测试error_handler模块的功能。每个测试阶段持续约7秒，循环执行：

1. **测试阶段1：OLED模块错误处理**
   - 测试无效行号（行号>4）
   - 测试无效列号（列号>16）
   - 测试空指针参数

2. **测试阶段2：LED模块错误处理**
   - 测试无效LED ID（LED_3不存在）
   - 测试无效LED ID（LED_0不存在）
   - 测试空指针参数（LED_GetState）

3. **测试阶段3：clock_manager模块错误处理**
   - 测试未初始化时的错误（如果可能）
   - 测试模式冲突（在自动模式下设置固定频率）

4. **测试阶段4：错误回调功能**
   - 注册错误回调函数
   - 触发错误，观察回调是否被调用（LED1闪烁）
   - 取消注册回调，再次触发错误，观察回调是否不再被调用

5. **测试阶段5：错误码转换功能**
   - 测试OLED错误码到字符串转换
   - 测试LED错误码到字符串转换
   - 测试CLKM错误码到字符串转换

6. **测试阶段6：ErrorHandler_Check功能**
   - 测试成功情况（无错误）
   - 测试错误情况（有错误）

### 关键方法

- **分阶段测试**：通过6个测试阶段，逐步测试不同模块和功能的错误处理
- **错误回调机制**：通过注册回调函数，在错误发生时执行自定义操作
- **错误码转换**：将错误码转换为可读的字符串，便于调试和显示

### 工作流程示意

```
系统初始化
    ↓
OLED初始化
    ↓
错误处理模块初始化
    ↓
测试阶段1：OLED模块错误处理
    ↓
测试阶段2：LED模块错误处理
    ↓
测试阶段3：clock_manager模块错误处理
    ↓
测试阶段4：错误回调功能
    ↓
测试阶段5：错误码转换功能
    ↓
测试阶段6：ErrorHandler_Check功能
    ↓
循环执行
```

## 📚 关键函数说明

### 错误处理相关函数

- **`ErrorHandler_Handle()`**：处理错误
  - 在本案例中用于手动处理错误，记录错误日志并调用回调函数
  - 参数：错误码、模块名称

- **`ErrorHandler_Check()`**：检查并处理错误
  - 在本案例中用于检查函数返回值，如果为错误则自动处理
  - 推荐使用方式，代码更简洁
  - 参数：错误码、模块名称
  - 返回值：1表示有错误，0表示无错误

- **`ErrorHandler_GetString()`**：获取错误码字符串
  - 在本案例中用于将错误码转换为可读的字符串
  - 用于OLED显示错误信息

- **`ErrorHandler_RegisterCallback()`**：注册错误回调函数
  - 在本案例中用于注册错误回调，错误发生时LED1闪烁
  - 回调函数中禁止嵌套调用ErrorHandler_Handle

- **`ErrorHandler_UnregisterCallback()`**：取消注册错误回调函数
  - 在本案例中用于取消注册回调，测试回调功能

### OLED相关函数

- **`OLED_ShowString()`**：显示字符串
  - 在本案例中用于显示错误信息（模块名称、错误码、错误描述）
  - 测试无效参数时会返回错误码

### LED相关函数

- **`LED_On()`** / **`LED_Off()`**：LED控制函数
  - 在本案例中用于错误提示（错误发生时LED1闪烁）
  - 测试无效LED ID时会返回错误码

### 时钟管理相关函数

- **`CLKM_SetFrequency()`**：设置固定频率
  - 在本案例中用于测试模式冲突错误
  - 在自动模式下设置固定频率会返回错误码

**详细函数实现和调用示例请参考**：`main_example.c` 中的代码

## ⚠️ 注意事项与重点

### ⚠️ 重要提示

1. **错误回调函数限制**：
   - ✅ **允许**：简单的LED闪烁、状态更新等快速操作
   - ❌ **禁止**：调用`ErrorHandler_Handle()`（会导致嵌套死锁）
   - ❌ **禁止**：执行阻塞操作（影响实时性）
   - ❌ **禁止**：在ISR中调用`ErrorHandler_Handle()`（除非标记为ISR-safe）

2. **错误处理方式**：
   - **方式1**：使用`ErrorHandler_Check()`（推荐，代码简洁）
   - **方式2**：手动调用`ErrorHandler_Handle()`（详细控制）
   - **方式3**：注册错误回调（自定义错误处理）

3. **测试循环**：
   - 6个测试阶段循环执行，每个阶段约7秒
   - 可以通过复位重新开始测试

4. **错误码格式**：
   - 错误码为负数，表示错误
   - 错误码为正数或0，表示成功
   - 每个模块有100个错误码空间

### 🔑 关键点

1. **统一错误处理**：
   - 所有模块函数返回统一的错误码类型（error_code_t）
   - 通过ErrorHandler统一处理错误，便于管理和调试

2. **错误回调机制**：
   - 可以注册错误回调函数，在错误发生时执行自定义操作
   - 回调函数必须遵守限制，避免嵌套调用和阻塞操作

3. **错误码转换**：
   - 通过ErrorHandler_GetString()将错误码转换为可读字符串
   - 便于调试和用户界面显示

### 💡 调试技巧

1. **错误处理不生效**：
   - 检查错误处理模块是否已启用（`CONFIG_MODULE_ERROR_HANDLER_ENABLED = 1`）
   - 检查是否正确调用ErrorHandler_Handle()或ErrorHandler_Check()

2. **回调函数不执行**：
   - 检查是否正确注册回调函数
   - 检查回调函数是否遵守限制（禁止嵌套调用、阻塞操作等）

3. **错误码转换失败**：
   - 检查错误码是否在有效范围内
   - 检查错误码定义是否正确

4. **OLED显示异常**：
   - 检查OLED连接和配置
   - 检查错误信息格式是否正确
