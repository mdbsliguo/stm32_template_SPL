# Net06 工程说明

W5500 + Paho Embedded C MQTT 客户端案例（小精灵 F103ZE）。

---

## 工程路径

```text
Examples/Net/Net06_W5500_Paho_Embedded_C/
  main_example.c          # 应用：tick 发布、命令订阅、Phy 监控
  board.h / board_early_init.* 
  startup_stm32f10x_hd.s  # 8KB 主栈（必用）
  Examples.uvprojx        # Keil 工程
  Doc/工程说明.md         # 本文件
```

中间件：

```text
Middlewares/protocols/paho_mqtt/
  paho_mqtt_client.c/h    # 封装：连接、订阅、离线 drain、pending 队列
  paho_platform.c/h       # W5500 Socket0 + Timer
  third_party/paho/       # Eclipse Paho 源码
```

---

## 编译与烧录

1. Keil 打开 `Examples.uvprojx`，Target 选 `Net06_W5500_Paho_Embedded_C`
2. **确认启动文件**为本目录 `startup_stm32f10x_hd.s`（8KB），不要用 `Start/` 下 1KB 版本
3. Rebuild → 烧录
4. 串口 **115200**，上电应看到 `=== Net06 W5500 Paho MQTT Client EXTI ===`

## 验收测试

### 1. 在线收发

- MQTTX 订阅 `Net06/stm32_out` → 每 5s 见 `tick=...`
- MQTTX 向 `Net06/stm32_in` 发 `hello`（QoS 1）→ 串口：

```text
[INFO ][MQTT] RX topic=Net06/stm32_in payload=hello
```

- 串口每 5s 见 `TX tick=...`

### 2. 离线小数据（拔网线）

1. MQTT 已连接
2. 拔网线 → `PHY link DOWN`
3. MQTTX 发 `1` `2` `3`（QoS 1，各 ≤128 字节）
4. 插网线 → `PHY link UP, recover net`
5. 串口：`drain offline...` → 多条 `RX topic=Net06/stm32_in payload=...`

### 3. 会话恢复

- 已连上过 Broker 后，**只按复位键**（不拔网线）
- 预期：`SessionPresent=1`、`session restored listen ...`（无需重新 SUBACK 等待）

---

## 限制

| 项 | 值 |
|----|-----|
| 单条 payload | ≤ 128 字节（`PAHO_MQTT_CLIENT_PAYLOAD_MAX`） |
| 离线 pending | 最多 32 条 |
| 主栈 | 8KB（`Stack_Size = 0x2000`） |

---

## 常见问题

| 现象 | 处理 |
|------|------|
| 串口乱码 / 死机 | 检查是否用了 8KB `startup_stm32f10x_hd.s` |
| 收不到离线消息 | MQTTX 须 **QoS 1**；Clean Session 关；Client ID 勿与板子冲突 |
| 同一数字 RX 很多次 | Broker 积压或重复发布；重启 Mosquitto 或清会话再测 |
| 无 `TX tick` | 等 MQTT 连接完成；offline drain 期间 tick 会暂缓 |
